fix mesh/stressanalysis command

Syntax: 

fix ID group-ID mesh/stressanalysis file filename type walltype mesh_keywords values stress_keywords values
  • ID, group-ID are documented in fix command
  • mesh/stressanalysis = style name of this fix command
  • file filename type walltype mesh_keywords values are documented in fix mesh.
  • zero or more stress_keyword/value pairs may be appended
  • stress_keyword= wear 
      wear value = finnie or none

Examples: 

fix cad all mesh/stressanalysis file mesh.stl type 1 finnie yes

LIGGGHTS vs. LAMMPS Info:

This command is not available in LAMMPS.

Description:

This fix is identical to fix mesh except for the fact that the pressure and shear force that the particles in the fix group exert on each triangle of the mesh is evaluated (which costs a bit of performance).

The forces can be dumped into VTK format using the following command: 

dump id group mesh/vtk N file stress

which is similar to the dump mesh/stl command except for the fact that a VTK file is written instead of an STL file which contains the STL data as well as pressure and shear stress. See dump mesh/vtk for details.

The optional wear keyword can activates a simple qualitative wear model (finnie) - for details on the model, see the seperate /doc/finnie-wear.pdf. The finnie constant k in Eqn. (4.23) has to be specified as follows: 

fix id all property/global k_finnie peratomtypepair n_atomtypes value_11 value_12 .. value_21 value_22 .. . 

    (value_ij=value for the finnie constant between atom type i and j; n_atomtypes is the number of atom types you want to use in your simulation)

Restart, fix_modify, output, run start/stop, minimize info:

This fix stores a global vector with 6 components for access by various output commands. The first 3 components are equal to the total force on the mesh, the last 3 components store the total torque on the body exerted by the particles. Other info see fix mesh.

Related commands:

fix wall/gran

Default:

finnie = no
